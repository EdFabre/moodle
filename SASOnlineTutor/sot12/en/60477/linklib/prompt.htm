The PROMPT | NOPROMPT Option

The PROMPT | NOPROMPT option modifies the effect of the INOBS= and OUTOBS= options so that you are prompted to stop or continue processing when the limits set by these options are reached. The default is NOPROMPT.


Example

In the following PROC SQL step, the PROMPT option causes a prompt window to appear after five rows (the limit set by the INOBS= option) are read from Sasuser.Internationalflights:

     proc sql inobs=5 prompt;
        select flightnumber, destination 
           from sasuser.internationalflights;




Selecting C to continue and OK causes the PROC SQL step to read another five rows from Sasuser.Internationalflights. After those rows are read, the prompt appears again.






Selecting S to stop and OK causes the PROC SQL step to stop processing. As indicated in the log, the resulting table contains 10 rows.
