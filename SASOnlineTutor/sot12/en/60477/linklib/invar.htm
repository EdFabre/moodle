Temporary IN= Variables

You can use the IN= data set option to exclude unmatched observations from a match-merge. Since the IN= variable is a temporary variable, it is available to program statements during the DATA step but it is not included in the SAS data set that is being created.


General form, IN= data set option:
(IN=variable)  

where

  • the IN= option, in parentheses, follows the data set name
  • variable names the variable to be created.

Within the DATA step, the value of the IN= variable is 1 if the data set contributed data to the current observation. If the data set did not contribute to the current observation, the value of the IN= variable is 0.


Example

Suppose you want to match-merge the Sasuser.Expenses data set with the Sasuser.Revenue data set. You use the IN= option to create two temporary variables whose values indicate whether the data set to which they are connected has contributed to the current observation in the Program Data Vector. Then you use a subsetting IF statement to include only observations that contain data from both input data sets.

In the following example, only those observations with data from both Sasuser.Expenses and Sasuser.Revenue are included in the output data set:

     data combined;
        merge sasuser.expenses(in=e) sasuser.revenue(in=r);
        by flightid date;
        if e and r;
     run;
