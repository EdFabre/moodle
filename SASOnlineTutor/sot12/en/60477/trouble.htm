Troubleshooting Tips

If you have questions about using these SAS programming lessons, courses, or libraries, check out these common solutions.

Click a link below to open a tip, and click the link again to close it.

Guided Practices and Examples

1.

Guided practice data sets do not exist

 

Solution: 

If SAS cannot find a data set that you referenced in a submitted SAS program, the SAS log displays a message indicating that the data set does not exist. Make sure that you have set up the sample data sets on your computer before you complete the guided practices. (You can identify guided practices by the words Try It! in the upper left corner.) To see the data setup instructions, do one of the following:

  • Select the link to Before You Begin on the home page for the lesson, course, or library. (To find out how to return to the home page, see tip 8 below.)

  • From any page within a lesson, select Help Before You Begin.

In addition, the first time that you open a guided practice page in a particular lesson, a Reminder pop-up window provides instructions for creating the sample data. You only need to create sample data once as long as you use the same computer and do not reinstall SAS (either the same version or another version).
 
2.

Guided practice filerefs are not assigned

 

Solution:

If SAS does not recognize a fileref that you referenced in a submitted SAS program, the SAS log displays a message indicating that the fileref is not assigned. In most lessons, the guided practices use only SAS data sets that are created when you first set up your sample data (see tip 1 above). These data sets are stored in the Sasuser and Work libraries, which SAS assigns by default.

However, in some lessons, the guided practices use raw data files. In each lesson that uses raw data files, the Overview page links to a SAS program that you must submit to set up the required filerefs in your operating environment. You must submit this program in each SAS session. In addition, the first time that you open a guided practice page in one of these lessons, a Reminder pop-up window provides instructions for setting up filerefs.
 
3.

Guided practice solution code generates unexpected results or errors

 

Solution:

If running the solution code does not give the expected results, check to make sure that

  • you typed or copied the solution code correctly.

  • you set up the required data sets (see tip 1 above).

  • in a guided practice that uses raw data files, you set up the required filerefs (see tip 2 above).

  • you completed any previous guided practices that must be completed before the current guided practice. When you must complete a previous guided practice, a note above the practice steps links back to the previous guided practice. When the previous and current guided practices use the same temporary data set, you must complete both guided practices in the same SAS session.
4.

Copied solution code looks different

 

Solution:

When you copy and paste code from a guided practice solution into SAS, the code indention may look different. In SAS, code indention is optional. The code executes correctly whether or not the lines are indented.
 
5.

Need to view the contents of a data set

 

Solution:

Sometimes, you need to view information about a data set in order to complete a guided practice. At the top of each guided practice that uses data sets, the practice description links to a descriptive pop-up page for each data set. Data set pop-up pages list the data set's variables and attributes and show representative sample data.
 
6.

Example data sets do not exist and/or filerefs are not assigned

 

Solution:

The SAS program code that is shown in examples (on lesson pages other than guided practice pages) is for illustrative purposes only. Much of the example code is based on fictitious data sets and is not intended to be executed. However, data sets are provided for all of the guided practices (see tip 1 above). For guided practices in lessons that require the use of raw data files, a program is provided to set up filerefs for the current session (see tip 2 above).
 

Navigation

7.

Moving to a different lesson in the current course or library

 

Solution:

To choose a different lesson at any time, select Path at the top of the page to return to the Learning Path page. The content of the Learning Path page varies, depending on whether you have licensed a single lesson, a course, or a library.

If you have licensed

  • a single lesson, the Learning Path page displays a list of lesson topics. (These lesson topics are also listed in the Lesson Contents drop-down menu at the top of each lesson page.) You will need to license any additional lessons that you want to access (see tip 11 below).

  • a course or a library, the Learning Path page displays all the lessons in the current course or library. From the list of lessons, select the lesson that you want.

Note: Use the (back) and (next) arrows at the top and bottom of each lesson page to navigate within the current lesson. These arrows cannot be used to exit the current lesson or to move to another lesson. In each lesson, the arrows are disabled on the first page and the arrows are disabled on the last page.
 

8.

Returning to the home page

 

Solution:

The home page is the first page that displays when you open the lesson, course, or library that you licensed. You must return to the home page if you want to reset your SAS version or to use the pop-up table of contents and index. To return to the home page from any lesson page, follow these steps:

  1. Select Path at the top of the page to return to the Learning Path page.

  2. Select Home at the top of the Learning Path page to return to the home page.
9.

Exiting a lesson, course, or library

 

Solution:

To exit a lesson, course, or library at any time, you can move to a different Web page in the same browser window, or you can close your browser. You don't have to do anything special to exit. When you return to the home page for the lesson, course, or library, you can click Resume to return to your previous place.
 

Miscellaneous

10.

Lesson does not match my version of SAS

 

Solution:

It is important to specify the version of SAS that you are using, because instructions and lesson content are tailored to a specific version of SAS. The first time you visit the home page, it prompts you to specify your SAS version. You can reset your SAS version at any time by returning to the home page (see tip 8 above) and clicking Reset your SAS version.
 
11.

Lessons referenced in training do not appear on the Learning Path page

 

Solution:

Some lesson pages contain references to related lessons that might be useful to you, even though they don't appear in the same course or library as the current lesson. To license additional lessons, courses, or libraries, or to view information about their contents, visit support.sas.com/selfpaced.
 
12.

Check mark does not appear on the Learning Path page

 

Solution:

This training keeps track of your progress by storing your progress information in a small text file (called a cookie). The cookie is created on your computer the first time you access the training. When you score 80% or higher on a quiz, your successful completion of the lesson is recorded in the cookie and a checkmark appears next to the lesson name on the Learning Path page.

To track your progress across multiple sessions, you must do both of the following:

  • Make sure that cookies are enabled in your browser.

  • Use the same computer to complete all of the lessons, courses, or libraries so that the cookie contains all of your progress information. If you switch to a different computer (or if the cookie is deleted), a new cookie will be created and your progress information will not be accessible in a single location.
