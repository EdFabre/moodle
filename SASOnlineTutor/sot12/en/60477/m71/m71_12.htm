Using Inner Joins (continued)

You are creating a PROC SQL query that uses an inner join to display data about students who have registered for classes. The data that you need is stored in two tables:

  • Sasuser.Register contains course data for all students who are currently registered for courses
  • Sasuser.Students contains company and address data for all students.

Student_Name is the only common column across tables, and the two Student_Name columns have an identical format. Your inner join combines rows from the two tables that have matching values of Student_Name, and displays only one of the Student_Name columns.

Add a FROM clause that specifies table aliases for the two tables. List the tables in either order, but be sure to use the alias r for Sasuser.Register and s for Sasuser.Students.

 

      proc sql;
        select r.student_name, student_company, 
               city_state, course_number, paid
                 
           where r.student_name=s.student_name;
