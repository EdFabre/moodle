Example: Viewing Resource Statistics for Windows

Suppose you submitted a PROC SORT step to sort the data set Work.A. The system option FULLSTIMER is in effect, so all the performance statistics of your computer system that are available to SAS are written to the SAS log:


SAS Log
NOTE: The data set WORK.A has 987792 observations 
and 19 variables.
NOTE: PROCEDURE SORT used (Total process time):
      real time           18.42 seconds
      user cpu time       23.42 seconds
      system cpu time     6.06 seconds
      Memory                            18066k


The SAS log displays a variety of resource usage statistics, including the following:


Statistic Description
real time

the amount of time spent to process the SAS job. Real time is also referred to as elapsed time.

Unless you are running SAS 9.1 with threading enabled, real time might be a less useful measure of program efficiency than CPU time.
user CPU time

the CPU time spent to execute your SAS code
system CPU time

the CPU time spent to perform operating system tasks (system overhead tasks) that support the execution of your SAS code
memory the amount of memory required for running a step


In the Windows operating environment, some statistics are not accurate unless the FULLSTIMER system option is specified at start-up time.

Different versions of the Windows operating system generate different statistics.

To gather more details about SAS resource usage in the Windows NT or Windows 2000 environment, you can use a Windows performance monitor. For more information, see the SAS documentation for the Windows operating environment.
