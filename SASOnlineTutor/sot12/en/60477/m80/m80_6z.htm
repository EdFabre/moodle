Example: Viewing Resource Statistics for z/OS

Suppose you submitted a PROC SORT step to sort the data set Work.A. The system options FULLSTATS, STIMER, MEMRPT, and STATS are all in effect, so the SAS log displays all available resource usage statistics:


SAS Log
NOTE: The data set WORK.A has 987792 observations 
and 19 variables.
NOTE: The PROCEDURE SORT used the following resources:
       CPU     time -         00:00:05.55
       Elapsed time -         00:00:48.77
       RSM Hiperspace  time - 00:00:00.76
       EXCP count   - 7116
       Task  memory - 200482K (197499K data, 2983K program)
       Total memory - 209007K (200096K data, 8911K program)


The SAS log displays a variety of resource usage statistics, including the following:


Statistic Description
CPU time

the actual time spent on the task by the CPU. This number should be constant (within .02 seconds) across repetitions of the same job.

elapsed time

the clock time (real time) required for completing the task
RSM Hiperspace the amount of CPU time that a SAS task uses to store data in a library that resides in a z/OS hiperspace. The value for RSM Hiperspace time is zero unless a hiperspace library is being accessed, and unless the HIPERSPACE system option has been issued.
EXCP count

the number of I/O operations required for transferring external data to and from memory. EXCP denotes EXecute Channel Program.
task memory

the actual memory, in kilobytes, required for a task, with breakdowns for data storage and program storage. This number is stable for a particular task.

total memory the memory, in kilobytes, required for all tasks. This session total is useful for deciding the minimum region size that the job requires in order to execute successfully.
