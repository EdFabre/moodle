Save a macro to an external file and use %INCLUDE to compile it.

  1. Copy the following program and paste it into the code editing window. Do not submit the program. 

    %macro prtlast;
   %if &syslast ne _NULL_ %then %do;
      proc print data=&syslast(obs=5);
         title "Listing of &syslast data set";
      run;
   %end;
   %else
      %put No data set has been created yet.;
%mend;
    Save the program as an external file named prtlast.sas. Note the location in which you save the file. You will need to include the path to the file in the next step.

  2. Open a new code editing window. Copy the following program and paste it into the code editing window. Add a statement after the PROC SORT step that will insert the contents of the external file that you created in Step 1. 

    proc sort data=sasuser.courses out=bydays;
   by days;
run;
    Submit the program.

  3. Submit a call to the Prtlast macro. Examine the output as well as any messages that are written to the SAS log.

  1. With the code window active, select FileSave filename As/Export. Navigate to the location to which you want to save the program. Enter prtlast.sas and click Save.

  2. Open a new Code window. Submit the following program. Be sure to replace my-location with the location of the file that you saved in Step 1. 
    proc sort data=sasuser.courses out=bydays;
    by days;
 run;
%include 'my-location\prtlast.sas';
     The PROC SORT step insures that the value of SYSLAST is not null The %INCLUDE statement compiles the Prtlast macro.

  3. Submit the following: 
     %prtlast
     Examine the output as well as any messages that are written to the SAS log. You should see the Work.Bydays data set in the output. Depending on the options that that you have specified, you might see messages about macro execution and macro variable resolution in the log.
