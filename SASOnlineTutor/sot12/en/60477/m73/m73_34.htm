Handling Errors in Row Insertions (continued)
Which of the following statements about the way in which PROC SQL handles insertion and update errors is true?
You can specify four settings for the UNDO_POLICY option: REQUIRED, NONE, OPTIONAL, ROLLBACK.
Specifying UNDO_POLICY=NONE overrides PROC SQL's UNDO processing, so all data, even data that violates established integrity constraints, is inserted or updated.
If you want PROC SQL to undo the insertion of all data if it finds any errors in the INSERT statement, you must specify the option UNDO_POLICY=OPTIONAL in your PROC SQL step.
When you try to insert data that violates a specified integrity constraint, the message in the SAS log identifies the VALUES clause that contains the problematic data.
