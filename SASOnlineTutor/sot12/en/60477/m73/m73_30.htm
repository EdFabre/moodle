Creating a Table That Has Integrity Constraints (continued)

Suppose you want to use a PROC SQL step to create the table Work.Departments, which will list all of a company's department names and codes. The table should have two columns: Dept and Code. You want to ensure that only unique department codes can be stored in the table.

Which of the following PROC SQL steps uses valid syntax to create Work.Departments with the integrity constraint described above?
  

proc sql;
   create table work.departments
          (Dept char(15),
          Code num label='Dept Code',
          constraint code_nodupe unique(code));
 
proc sql;
   create table work.departments
          (Dept char(15),
          Code num label='Dept Code'
          constraint code_nodupe unique);
 
proc sql;
   create table work.departments
          (Dept char(15),
          Code num label='Dept Code',
          constraint unique(code));
 
proc sql;
   create table work.departments
          (Dept char(15),
          Code num label='Dept Code',
          constraint code_unique=code);
