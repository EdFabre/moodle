Preparing Data for Reports (continued)

Review of Using the Query Builder to Create Data

In addition to simply adding data to a project, you often need to manipulate your data before it is ready to use for reports. As you saw in the Getting Started tutorial, you can use the Query Builder to join tables, select and delete columns, create new columns, and subset the data.





When you are ready to create a report from a query, you can use any of the following as input data:

  • a data set that is generated from the query
  • a data view that is generated from the query
  • the query itself.

By using the query or a data view that is created from the query as input to the reporting task, you can run the task, and the query will run before the task runs. If your data has changed, those changes are reflected in the report. However, if you use an output data set as input data and your data has changed, you must first rerun the query to re-create the output data set.

To learn more about running queries, see the lesson .
