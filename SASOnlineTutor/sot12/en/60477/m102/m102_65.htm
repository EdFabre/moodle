Using Traffic Lighting in a Summary Table

Modifying Code in SAS Enterprise Guide

The tasks in SAS Enterprise Guide are based on SAS procedures in the SAS programming language. As you assign variables and select options in the task window, SAS Enterprise Guide writes a SAS program to submit to SAS. Each procedure can use many options, and the most common options are available in the task window. However, there are certain things that you can do only by modifying the SAS program that a task creates.

SAS Enterprise Guide enables you to modify the code that is generated by a task and to submit that code to SAS. After you modify the code, a code item is added to the project and it is completely separate from the task. If you make changes by using the task window, those changes are not included in the code you have modified.





Modifying Code to Add Traffic Lighting

The Summary Tables task uses the TABULATE procedure. In the TABULATE procedure code, you can add a STYLE= option to a variable or statistic in a table. You can use the STYLE= option to change the background color of a table cell, fonts and font attributes, and many other aspects of the style.

You might remember that when you created a summary table report, you specified heading appearance options in the task window. The general format for crossing the STYLE= option with a variable or statistic in a TABLE statement is as follows:

    variable-or-statistic*{style={attribute=value}}

When you created a summary table report of profit by category, the task generated PROC TABULATE code that included the style option on some of the variables and statistics.






The code for adding a background color that is based on a format is as follows:

    *{style={background=format.}}

Next, you create a format that will change the background cell color based on a range of variable values.
