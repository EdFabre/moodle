Storing and Accessing Code

Exporting a Copy of Your Code to a File

The code that you create in SAS Enterprise Guide is embedded in the project. This means that the code is actually part of the project (.SEG) file. When you save the project, you also save the code that you have written. You can open the code by reopening the project and double-clicking the code in the Project window.

If you want the code to be saved separately from the project, you must export the code. This saves the code as a text file in the location that you specify. To save a copy of the code as a .SAS file, you select the code and use the File menu or the pop-up menu to select Save filename As/Export.





After you export a copy of your program, the code that you open from the Project window is still embedded in the project and editable. Changes to the code in the project do not affect the code that you exported to a file.

You can use FileExport All Code to export all the code in a project to a single file.


Creating a Pointer to an External Code File

If you want to save code to a file and also edit that file when you open the code item in the Project window, you can change the properties of the file. To change the properties, right-click the code and select Properties from the pop-up menu. Then click the Save As button. After you specify a file location and click OK, the item in the Project window becomes a shortcut to the code file and the code is no longer embedded in the project.
