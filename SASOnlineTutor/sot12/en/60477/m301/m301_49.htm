Use a the Query Builder to generate SQL code and then edit the code.

  1. Start a query on the Houses data, which is located in the SAS Enterprise Guide sample data folder.

    • Select ToolsQueryCreate Empty.
    • Click Add Data.
    • Navigate to the sample data directory and double-click Houses.sd2.
  2. Replace values of the STYLE column so that both TWOSTORY and SPLIT are replaced by Multi-level.

    • Click the Select and Sort tab.
    • Select the STYLE column and click Properties.
    • Click the Replace Values tab.
    • From the Replace drop-down list, select TWOSTORY.
    • In the Replace with box, type Multi-Level.
    • Click Add to list.
    • From the Replace drop-down list, select SPLIT.
    • Click Add to list.
    • Click OK.
  3. Run the query. Be sure to close the output data set from the query.

    • Click Run Query.
    • Close the output data set.
  4. Edit the query code to add else houses.style to the CASE statement so that current values of the STYLE column are used when the value is not replaced.

    • In the Project window, right-click Query1 and select Add as Code Template.
    • Add else houses.style to the CASE statement as follows: 
      (CASE Houses.STYLE 
   WHEN 'TWOSTORY' THEN 'Multi-Level'
   WHEN 'SPLIT   ' THEN 'Multi-Level'
   else houses.style
 END) AS _STYLE_minus_,
  5. Run the query code.

    • Press F3.
    • Notice that the data now includes the values for CONDO and RANCH in the _STYLE_minus_ column.

  1. Start a query on the Houses data, which is located in the SAS Enterprise Guide sample data folder.

    • Select DataCreate New Empty Query.
    • Click Add Data.
    • Navigate to the sample data directory and double-click Houses.sd2.
  2. Keep only the BEDROOMS and PRICE columns in the query results.

    • Click the Select and Sort tab.
    • Press Ctrl and select STYLE, SQFEET, and BATHS.
    • Click Delete.
  3. Run the query and view the results.

    • Click Run Query.
    • Take a look at the results. Suppose you want to add a note to any home that has only one bathroom.


  4. Edit the query code to add a Notes column that contains a message when the home has only one bathroom. Use the following line of code:

    (CASE Houses.baths WHEN 1 THEN 'Warning: single-bath house' END) AS Notes

    • In the Project window, right-click Query1 and select Add as Code Template.
    • Add a comma after
      Houses.PRICE FORMAT=DOLLAR12.
      and then add the CASE statement shown above.
  5. Run the query code.

    • Right-click Code for Query1 for Houses and select Run on Local from the popup menu.
    • Notice that the Notes column contains a message for every home that has only one bathroom.
