Validating Query Syntax (continued)
The following statements describe the NOEXEC option and the VALIDATE keyword. Which statement is false?
The NOEXEC option and the VALIDATE keyword are both used to check the validity of PROC SQL queries without executing them.
The NOEXEC option is specified in the PROC SQL statement and the VALIDATE keyword appears just before the SELECT statement.

To validate multiple SELECT statements within a single PROC SQL query, you specify the NOEXEC option one time. However, you must specify the VALIDATE keyword prior to each SELECT statement.
When a PROC SQL query that contains the NOEXEC option or the VALIDATE keyword is run, the SAS log indicates only whether the query is valid. The usual SAS error messages are not displayed.
