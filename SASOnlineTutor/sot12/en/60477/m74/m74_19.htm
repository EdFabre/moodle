Monitor and control the use of the index LastName on the temporary table Work.Staffmaster (which is a copy of Sasuser.Staffmaster).

You must complete the previous practice before you can do this practice.

  1. Paste the following PROC SQL query into your code editing window. Add a statement that contains the SAS option that will display informational notes about index usage in the SAS log. Submit the program and view the SAS log to see whether the index LastName was used to process your query.

    proc sql;
   select *
      from work.staffmaster
      where lastname contains 'AR';
quit;
  2. Modify the previous query as follows:

    • Remove the statement that contains the SAS option that displays informational messages in the SAS log.

    • Add a SAS data set option to force PROC SQL to process the query without using the index.

    Submit this modified query and view the SAS log to verify that SAS did not use the index in processing.

  3. Submit a statement that resets the SAS option MSGLEVEL= to the default setting. (By default, the SAS log displays notes, warnings, and error messages only.)
