Managing Index Usage (continued)

You are working with the temporary table Weather, which has two indexes:

  • the simple index Region that is defined on the column Region
  • the composite index named RegionType, which is based on the columns Date (the primary key) and Region (the secondary key).

Which PROC SQL step forces SAS to use the index Region? 


proc sql;
   select date,stormid,name,type,severity,region
      from work.weather(idxname=region)
      where region in ('04','06','17');
 
proc sql;
   select date,stormid,name,type,severity,region
      from work.weather(idxname='region')
      where region in ('04','06','17');
 
proc sql idxname=region;
   select date,stormid,name,type,severity,region
      from work.weather
      where region in ('04','06','17');
 
proc sql;
   select date,stormid,name,type,severity,region
      from work.weather(idxwhere=region)
      where region in ('04','06','17');
