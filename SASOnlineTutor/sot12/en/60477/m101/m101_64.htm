Open a query and change the join type to see how the query output is affected.

This practice uses the Practice1 query that was created in a previous practice. If you haven't created this query, return to the practice and create it before continuing here.

  1. Reopen Practice1.

    • In the Project window, double-click Practice1.
  2. Add the ListingSales table to the query and manually join the tables.

    • Click Add Data. Navigate to the directory where you stored the downloaded data. Double-click ListingSales. Click OK when you are prompted to join the tables manually.
    • In the Listings table, click PROPID and drag the join line to the PropertyID column in the ListingSales table.
  3. Include only the following columns (in order) in the query results: PROPID, PRICE, SalesPrice, PropertyID.

    • Click the Select and Sort tab. Click Select All, then click Delete.
    • From the list of columns for the Listings table, double-click PROPID, then PRICE.
    • From the list of columns for the ListingSales table, double-click SalesPrice, then PropertyID.
  4. Preview the results. How many properties are included in the results?

    • Click Preview Results.
    • There are 10 properties with matching property identification numbers.

  5. Change the join type to a left outer join, then preview the results. Are there additional properties listed in the results?

    • Click the Tables tab.
    • Right-click the join symbol and select Modify Join.
    • Select All rows from Listings. Click OK.
    • As you might expect, there are properties that have not sold. With this join, the results include all properties, including those that are still on the market.
  6. Change the join type to a right outer join, then preview the results. What changes do you notice in the results?

    • Right-click the join symbol and select Modify Join.
    • Select All rows from ListingSales. Click OK.
    • There are properties listed from the ListingSales table that had no match in the Listings table. These properties might need to be added to the Listings table, or they might have incorrect PropertyID numbers.

  7. Change the join type to a full outer join, then preview the results.

    • Right-click the join symbol and select Modify Join.
    • Select All rows from Listings and ListingSales. Click OK.
    • The results include all rows from both tables.
  8. Run the query and do not replace the previous results. Rename the query Practice7.
