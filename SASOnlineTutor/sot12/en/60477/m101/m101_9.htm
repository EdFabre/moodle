Working with Columns in a Query

Now that you've started your query, let's see how you need to work with columns to determine inventory levels.

By default, all columns from all tables appear in your output, in the order in which they appear in the tables. For your report, you'll need to

  • select existing columns
  • compute new columns from other columns
  • order columns
  • sort rows by values of a column.


Example: Using a Query to Check Inventory

Using the Products table, you want to display these columns in your report:

  • ProductName
  • UnitsInStock
  • UnitsOnOrder
  • ReorderLevel.

You also want to compute two new columns, Total Inventory and Total Ordered, to show the dollar amount of products in inventory and products on order, respectively.

Remember that this is the report you want to generate:



Click to view the full window.


Notice that the new dollar amount columns follow the related units columns. Also, rows are sorted by dollar amount for total inventory, with the largest values first.

Now try working with columns in the Query Builder.
