Using Outer Joins (continued)

Now let's see how you can use outer joins to combine the Products and ProdCategories tables.

You've used the Products table in several queries already. For example, in the Group-Profit query, you grouped products by category. However, in your query output, you saw only the category ID, not category names.

The ProdCategories table contains both the category ID and the category name (CategoryName) for the categories. By joining the Products and ProdCategories tables by CategoryID, you can show the category names instead of the category IDs in your query output. Showing the descriptive information makes your query output easier to interpret and more meaningful.
