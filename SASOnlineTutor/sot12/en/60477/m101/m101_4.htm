Specifying a Result Format

After you complete all the specifications for your query in the Query Builder, you can generate your results. As you'll see below, you have several options for generating query output.

Choosing a type of output in advance ensures that your query provides information in the appropriate format. For example, to view the results of your queries for the specialty foods shop in your Web browser, you can generate output as HTML reports.


Types of Query Output

You can generate your query results in any of these formats:

  • a data table
  • a data view
  • a report.

Each of these types of output has advantages, as the following table shows.


Result Format

Characteristics of Data

 Query Results
data table

stored permanently

 

A static data table against which you can run SAS Enterprise Guide tasks.

Query results in this format are not updated until you rerun the query.

data view

always up to date

 

A dynamic data view against which you can run SAS Enterprise Guide tasks.

Each time you open query results in the data view format, the results are updated with any changes to the data that is used in the query.
report

formatted for presentation

 

A report in the format that you have specified as your default (HTML, PDF, RTF, or text).

Query results in this format are not updated until you rerun the query. You cannot run SAS Enterprise Guide tasks against query results in this format.


Understanding Data Views

You may be more familiar with data tables and reports than with data views. While data tables and reports actually contain data values, a data view contains only the logic for retrieving data that is stored elsewhere—not the data itself. Data views allow you to read data dynamically without using disk space to create a new data table. You can think of a data view as a stored query that you can use as data for Enterprise Guide tasks.

Data views are useful because they

  • often save space (a view is usually quite small compared with the data that it accesses)
  • ensure that input data sets are always current, because data is derived from tables when you use the data view.

If you share your data views, they are also useful in

  • shielding sensitive or confidential columns from users while enabling the same users to view other columns in the same table
  • hiding complex joins or queries from users.

Selecting a Type of Query Output

You can specify a default format for all query output, and you can override this setting for individual queries. To choose a result format, you can use

  • the Options window to set a default result format for all query output
  • the Query Builder or the project tree to specify the result format for a specific query.
