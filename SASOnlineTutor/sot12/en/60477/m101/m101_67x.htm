1.  

 Which result format(s) can you run SAS Enterprise Guide tasks against?

a. only data tables
b. data views and reports
c. only data views
d. data tables and data views


You can run SAS Enterprise Guide tasks against both data tables and data views. While data tables actually contain data values, a data view contains only the logic for retrieving data that is stored elsewhere—not the data itself. Query results in the form of reports do not store data in a form that SAS Enterprise Guide tasks can read. 


2.  

 When you set a filter, you can specify

a. only one filter condition.
b. multiple filter conditions, but no logical conditions (AND or OR).
c. multiple filter conditions and logical conditions (AND or OR).
d. none of the above


In filters, you can specify multiple filter conditions. When you specify more than one filter condition, you must specify its logical relationship (AND or OR) to other conditions.

4.  

 You can join tables manually only if the tables contain a related column with

a. the same name.
b. the same data type.
c. the same name and data type.
d. none of the above


The common column that you use to manually join tables must have the same data type in all tables, but it may have a different name. In fact, having matching columns with the same data type but different names is often what requires you to join tables manually.

6.  

 When you replace values, you can

a. replace a range of values with an individual value.
b. replace an individual value with another value.
c. replace an individual value with one value and replace all other values with another value.
d. all of the above


In the Query Builder, you can specify values to replace individual values or ranges of values for a column. You can also specify a value for all other values of a column. 


7.  

 Which statement about parameters is false?

a. The only type of parameter that you can specify is an item from a list.
b. When you create a parameter, you must also create a filter that uses the parameter.
c. You can create a query that accepts a parameter in such a way that you specify a value for the parameter whenever you run the query.
d. Using a parameter with a filter in your query makes the filter dynamic.


When you create a parameter, you can choose from several types of parameters, such as character strings, numbers, items in a list, dates, and times. You can even allow multiple selections from a list. 


8.  

Suppose that the Select and Sort tab in the Query Builder lists 6 columns. Three of the listed columns are selected. The first selected column is ProductName and values in the column are sorted in descending order with a sort priority of 2. The second selected column is _Country and values in the column are sorted in ascending order with a sort priority of 1.

If you run the query, what does the output display?

a. six columns, with _Country as the first column and rows sorted by descending values of ProductName
b. six columns, with ProductName as the first column and rows sorted by descending values of ProductName and by ascending values of_Country
c. three columns, with ProductName as the first column and rows sorted first by descending values of ProductName, then by ascending values of_Country
d. three columns, with ProductName as the first column and rows sorted first by ascending values of_Country, then by descending values of ProductName


Only three columns are selected to display in output, and they are displayed in the order shown (ProductName, _Country, and sum_of_QUANTITY). The rows are sorted first by ascending values of _Country because its sort priority is 1. Then rows are sorted by descending values of ProductName.
