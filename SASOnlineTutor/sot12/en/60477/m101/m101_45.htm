Create a parameterized query so that the house style can be selected when the query runs.

This practice uses the Practice1 query that was created in a previous practice. If you haven't created this query, return to the practice and create it before continuing here.

  1. Reopen Practice1.

    • In the Project window, double-click Practice1 to open the query.
  2. Include the following columns (in order) in the results: STYLE, BEDROOMS, BATHS, SQFEET, and PRICE.

    • Add and delete columns as needed. Select columns and click Move Up or Move Down to reorder columns.

  3. Create a new parameter named style that allows the user to select from a list of home styles.

    • Click the Parameters tab.
    • Click the Type of parameter drop-down list and select Item from a list.
    • In the Name box, type style.
    • Type any text prompt in the Message for user box.
    • Click Fill List with Actual Data Values.
    • Select STYLE and click OK.
    • Click OK to create the new parameter.

  4. Add a filter that selects rows where the value of the STYLE column is equal to the value in the &style macro variable.

    • Click the Filter Data tab.
    • Drag the STYLE column to the filter area.
    • In the Value box, type &style.
    • Click OK to set the filter.
  5. Run the query and do not replace the previous results. Examine the results. Do the results show only the style of home you chose in the parameter window?

    • Click Run Query. Select any home style in the parameter window. When you are prompted to overwrite the previous results, click No.
    • The Parameter window reopens. Select SPLIT.
    • Only split level homes are listed.
  6. Rename the query Practice5.
