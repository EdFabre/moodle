DATA Step Processing of Records That Have a Varying Number of Fields

This animation shows the processing of a DATA step that reads the varying number of fields in the raw data file Bpdata.

Raw Data File Bpdata
1---+----10---+----20---+----30---+----40---+----50
1234 13MAR89 120/80
1443 12FEB89 120/70 03FEB90 125/80 07OCT90 125/99
1681 11JAN90 120/80 05JUN90 110/70
2034 19NOV88 130/70 12MAY89 150/90 23MAR90 130/80

  1. The following program is submitted to SAS: 
    data perm.health;
       infile bpdata length=reclen;
       input ID 4. @;
       do index=6 to reclen by 15;
          input Date : date. BP $ @;
          output;
       end;
     run;
    The first time the DATA step executes, the first INPUT statement reads the values for ID and holds the record. Then the numeric length of the first record (reclen) is assigned.

  2. The initial value of index is 6, so the loop begins to execute.

  3. The second INPUT statement reads the first block of values for Date and BP, and holds the record.

  4. The OUTPUT statement writes the values to the data set as the first observation.

  5. The END statement passes control back to the DO statement, where the value of index is incremented by 15 to a total of 21. Now index exceeds the value of reclen, and the loop is not executed.

  6. Control returns to the beginning of the DATA step. The values in the program data vector are reset to missing, and the input pointer moves down to column 1 of the next record.
