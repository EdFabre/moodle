Reading Records That Have a Varying Number of Fields (continued)

The first three lines of the DATA step are similar to the previous example.

 

     data perm.health;    
        infile bpdata length=reclen;
        input ID 4. @;
The LENGTH= option creates a variable named reclen, whose value is the length of the current record. Later you can use reclen to specify how many times to execute an INPUT statement that reads the values of the dates and blood pressure readings.

The trailing @ line-hold specifier at the end of the INPUT statement holds the current record so that a later INPUT statement can read the values for Date and BP.
