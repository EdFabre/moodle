Conversion of SAS Variable Names to DBMS Column Names

Current SAS Variable Name Desired DBMS Column Name Required LIBNAME Option
any SAS variable name, such as Miles default DBMS column name (following DBMS naming conventions), such as MILES preserve_col_names=no
case-sensitive SAS variable name, such as Miles case-sensitive DBMS column name, such as Miles preserve_col_names=yes

If PRESERVE_COL_NAMES=YES, then special characters and the exact, case-sensitive spelling of the names are preserved when column names are read from and passed to the DBMS. In order to use column names that are not valid SAS names in your SAS program, you must use either the DQUOTE=ANSI option or the VALIDVARNAME=ANY option. To learn more about these options, see the SAS/ACCESS documentation.
