Processing SQL Statements

Although SAS can efficiently process DBMS data through the LIBNAME engine, sometimes you might want to have explicit control over the SQL statements that are passed to the DBMS. In some cases, the DBMS can process a query more efficiently. SAS/ACCESS software provides the SQL Procedure Pass-Through Facility, an extension to PROC SQL that enables you to send DBMS-specific SQL statements directly to a DBMS.

The SQL Procedure Pass-Through Facility provides the statements and component that are shown in the table below.


Statement or Component Function
CONNECT statement establishes a connection to the DBMS
CONNECTION TO component
(in the FROM clause of the PROC SQL SELECT statement)		 retrieves data directly from the DBMS
EXECUTE statement sends dynamic, non-query, SQL statements to the DBMS
DISCONNECT statement terminates the connection to the DBMS


You can embed Pass-Through statements in a PROC SQL query, or you can store them in a PROC SQL view. This tutorial focuses on embedding Pass-Through statements in a query.


Sample Program

Before we examine the SQL Procedure Pass-Through Facility in detail, let's look at a sample program. The PROC SQL step shown below does the following:

  1. connects to an Oracle database and assigns the alias Dblink to it
  2. uses this connection to read data from an Oracle table named EMPLOYEES
  3. grants UPDATE and INSERT authority for the table to the user
  4. disconnects from the Oracle database. 


  

    
 
      
 

      
proc sql;

    

    
 
       1 
      
   connect to oracle as dblink
           (user=cestmoi pw=trustme path=orapath
            connection=global);

    

    
 
       2 
      
   select *
      from connection to dblink
         (select id, lastname, firstname,
                 hiredate, salary
             from employees
             where hiredate>='31-DEC-99');

    

    
 
       3 
      
   execute (grant update, insert on employees
            to cestmoi) by dblink;

    

    

       4 
      
   disconnect from dblink;

    

    
 
       
 

      
quit;
