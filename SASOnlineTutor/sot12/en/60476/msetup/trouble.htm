Troubleshooting Tips

If you have questions about using these SAS programming lessons, courses, or libraries, check out these common solutions.

Click a link below to open a tip, and click the link again to close it.

Guided Practices and Examples

1.

Guided practice data sets do not exist

 

Solution: 

If SAS cannot find a data set that you referenced in a submitted SAS program, the SAS log displays a message indicating that the data set does not exist. Make sure that you have set up the sample data sets on your computer before you complete the guided practices. (You can identify guided practices by the words Try It! in the upper left corner.) To see the data setup instructions, select Before You Begin.

In addition, the first time that you open a guided practice page in a particular lesson, a Reminder pop-up window provides instructions for creating the sample data. You only need to create sample data once as long as you use the same computer and do not reinstall SAS (either the same version or another version).
 
2.

Guided practice filerefs are not assigned

 

Solution:

If SAS does not recognize a fileref that you referenced in a submitted SAS program, the SAS log displays a message indicating that the fileref is not assigned. In most lessons, the guided practices use only SAS data sets that are created when you first set up your sample data (see tip 1 above). These data sets are stored in the Sasuser and Work libraries, which SAS assigns by default.

However, in some lessons, the guided practices use raw data files. In each lesson that uses raw data files, the Overview page links to a SAS program that you must submit to set up the required filerefs in your operating environment. You must submit this program in each SAS session. In addition, the first time that you open a guided practice page in one of these lessons, a Reminder pop-up window provides instructions for setting up filerefs.
 
3.

Guided practice solution code generates unexpected results or errors

 

Solution:

If running the solution code does not give the expected results, check to make sure that

  • you typed or copied the solution code correctly.

  • you set up the required data sets (see tip 1 above).

  • in a guided practice that uses raw data files, you set up the required filerefs (see tip 2 above).

  • you completed any previous guided practices that must be completed before the current guided practice. When you must complete a previous guided practice, a note above the practice steps links back to the previous guided practice. When the previous and current guided practices use the same temporary data set, you must complete both guided practices in the same SAS session.
4.

Copied solution code looks different

 

Solution:

When you copy and paste code from a guided practice solution into SAS, the code indention may look different. In SAS, code indention is optional. The code executes correctly whether or not the lines are indented.
 
5.

Need to view the contents of a data set

 

Solution:

Sometimes, you need to view information about a data set in order to complete a guided practice. At the top of each guided practice that uses data sets, the practice description links to a descriptive pop-up page for each data set. Data set pop-up pages list the data set's variables and attributes and show representative sample data.
 
6.

Example data sets do not exist and/or filerefs are not assigned

 

Solution:

The SAS program code that is shown in examples (on lesson pages other than guided practice pages) is for illustrative purposes only. Much of the example code is based on fictitious data sets and is not intended to be executed. However, data sets are provided for all of the guided practices (see tip 1 above). For guided practices in lessons that require the use of raw data files, a program is provided to set up filerefs for the current session (see tip 2 above).
 

Miscellaneous

7.

Lesson does not match my version of SAS

 

Solution:

It is important to specify the version of SAS that you are using, because instructions and lesson content are tailored to a specific version of SAS. The first time you open a course or library, it prompts you to specify your SAS version. You can reset your SAS version at any time by returning to Before You Begin and clicking Reset your SAS version.
 
8.

Lessons referenced in training do not appear in the course or library

 

Solution:

Some lesson pages contain references to related lessons that might be useful to you, even though they don't appear in the same course or library as the current lesson. To license additional lessons, courses, or libraries, or to view information about their contents, visit support.sas.com/selfpaced.
