Summary of Match-Merge Processing

This animation summarizes the the compilation and execution phases of a match-merge.

  1. The following program is submitted to SAS: 
    data work.claims
       merge work.clients work.amounts;
   by name;
    run;
  2. First, SAS sets up the new data set by reading the descriptor portions of the data sets and creating the PDV.

  3. Next, SAS sequentially match-merges observations and writes the new observation to the PDV, then to the new data set.

  4. When the BY value changes in all the input data sets, the PDV is initialized to missing.

  5. Missing values for variables, as well as missing values resulting form unmatched observations, are written to the new data set.
