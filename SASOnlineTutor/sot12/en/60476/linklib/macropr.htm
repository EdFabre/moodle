Macro Processing

  1. The input stack contains the following program: 
         %let amt=1.1;
     data new;
        set perm.mast;
        bonus=wage*&amt;
     run;
  2. The word scanner reads the first two tokens of the %LET statement, and it passes that trigger to the macro processor.

  3. The macro processor requests additional tokens until it receives a semicolon.

  4. The macro processor creates a macro variable named amt in the global symbol table and assigns a value of 1.1 to it.

  5. The macro processor gives control back to the word scanner. The word scanner passes the next few statements to the compiler until it reaches the next macro trigger.

  6. The word scanner passes &amt to the macro processor.

  7. The macro processor looks up amt in the global symbol table and sends the value of amt (1.1) back to the input stack.

  8. Word scanning continues, but since the reference has been resolved to its value, the word scanner passes the token to the compiler.

  9. When SAS detects the RUN statement, which is a step boundary, the step executes.
