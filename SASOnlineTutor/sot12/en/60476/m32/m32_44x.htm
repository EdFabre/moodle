1.  

 Macro variables can be defined and referenced anywhere in a SAS program except within...

a. data lines
b. procedures
c. the FORMAT statement
d. the INPUT statement


Macro variables can be used anywhere in a SAS program except within data lines. If your DATA step reads instream data (data that is included in the program), you cannot use a macro variable within those lines. 

2.  

 Which assignment statement will set the value of Day to a text string that contains the day of the week?

a. Day=sysday;
b. Day=&sysday;
c. Day='&sysday';
d. Day="&sysday";


Double quotation marks are needed in order to correctly resolve the text string that is contained in the macro variable sysday. 


3.  

 Which statement creates a macro variable named region that has the value northwest?

a. %let region=northwest;
b. %let region='northwest';
c. &let region=northwest;
d. &let region="northwest";


You use the %LET statement to create a macro variable. The macro variable value is not enclosed in quotation marks. If quotation marks are used, they are stored as part of the value. 


4.  

 What is the result of including the following line in your program?

options symbolgen;

a. Messages about macro variable resolution are listed in program output.
b. Messages about macro variable resolution are suppressed in program output.
c. Messages about macro variable resolution are written to the SAS log.
d. Messages about macro variable resolution are suppressed in the SAS log.


The SYMBOLGEN system option causes messages about macro variable resolution to be written to the SAS log. These messages can help you verify the values of macro variables. NOSYMBOLGEN (the default) suppresses these messages. 


5.  

 When SAS encounters a macro variable reference but cannot locate the macro variable, the reference is not resolved. What happens next?

a. SAS continues execution, ignoring all lines that contain the macro variable reference.
b. SAS continues execution, but the unresolved reference often results in a syntax error.
c. SAS continues execution, but the unresolved reference often results in a system crash.
d. SAS ends program execution.


SAS continues scanning, but an unresolved macro variable reference often results in a syntax error. These errors often occur because the ampersand that appears in an unresolved macro variable reference cannot be used in most SAS statements. 

6.  

 Suppose your program creates a macro variable named endtime whose value is End. How would you use this macro variable to reference the variable EndDate in the program?

a. %endtime&Date
b. &endtime%Date
c. &endtime.Date
d. "%endtime"Date


When combining a macro variable prefix and a suffix, separate them with a period. Otherwise, SAS has no way of knowing where your macro variable reference ends and the suffix begins. 


7.  

 Suppose your program creates a macro variable named libref that contains the value sales. Another macro variable, named dsname, contains the value q3y2001. How would you use these macro variables to reference the data set Sales.Q3y2001?

a. data %libref.%dsname;
b. data &libref..&dsname;
c. data %libref".%dsname";
d. data "&libref."&dsname;


If your macro variable reference must appear before a suffix that begins with a period, you must specify two periods between the macro variable reference and the suffix. The first period ends the macro variable reference. 


8.  

 You use the CALL SYMPUT routine to create a macro variable whose value is assigned...

a. by a later statement
b. by another macro variable
c. during the execution of the DATA step
d. during the execution of a procedure


The CALL SYMPUT routine enables you to assign a value that is produced during DATA step execution to a macro variable. 


9.  

 Why is a PUT function embedded in this CALL SYMPUT statement?

call symput('total',put(totover,2.));

a. to ensure the conversion of numeric values to character values
b. to prevent the conversion of numeric values to character values
c. to resolve the macro variable reference of totover
d. to remove leading blanks from the values that are assigned to total.


Numeric values are automatically converted to character values as the values are assigned to the macro variable. This automatic conversion creates leading blanks. You can remove the blanks by using the PUT function. 


10.  

 What is wrong with the following program? 
title "Temporary Employees Worked &total OT Hours";
     data hrd.overtime;
        set hrd.temp(keep=name overtime) end=last;
        if overtime ne .;
        TotOver+overtime;
        if last then call
           symput('total',put(totover,2.));
     run;
     proc print data=hrd.overtime;
     run;

a. The PUT function is in the wrong place.
b. The TITLE statement is in the wrong place.
c. The CALL SYMPUT statement syntax is incorrect.
d. The CALL SYMPUT statement cannot be combined with an IF statement.


You cannot reference the macro variable in the TITLE statement because it precedes the DATA step in which the macro variable is created. The TITLE statement must be moved below the DATA step but before the PRINT procedure.
