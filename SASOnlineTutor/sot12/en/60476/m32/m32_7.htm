Using Automatic Macro Variables (continued)

You might have noticed that the FOOTNOTE statement text that includes the macro variable reference is enclosed in double quotation marks. Macro variable references that appear in quoted strings must be enclosed in double quotation marks.

If single quotation marks enclose a macro variable reference, SAS does not resolve the reference. 

     title 'Temporary Employees Hired in November';
     footnote "Report Run on &sysdate";
     data hrd.tempnov;
        set hrd.temp;
        if month(begindate)=11;
     run;
     proc print data=hrd.tempnov;
     run;

Here's an example. The program below encloses the macro variable reference in single quotation marks. When you submit the program below, you see an unresolved macro variable reference in the footnote. 

     title 'Temporary Employees Hired in November';
     footnote 'Report Run on &sysdate';
     data hrd.tempnov;
        set hrd.temp;
        if month(begindate)=11;
     run;
     proc print data=hrd.tempnov;
     run;

Report Run on &sysdate
