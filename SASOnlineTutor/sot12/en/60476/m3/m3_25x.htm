1.  

 Which of the following editors can you use to modify this existing SAS program? 
     data sasuser.mysales2; 
        set sashelp.prdsale; 
        if region="EAST";
     run;

a. the host editor of your choice
b. the SAS Notepad window
c. the Program Editor window
d. all of the above


You can use any of several types of editors to create and edit SAS programs. In the Windows operating environment, you can also use the enhanced Editor window. You can submit programs directly from the Program Editor, enhanced Editor, and SAS Notepad windows. In the Program Editor window, your code disappears after it is submitted.  

2.  

 Which SAS windows perform these tasks (in the order given)?

  • perform most file management tasks such as moving, copying, and deleting files
  • navigate and manage output from SAS programs
  • enter, edit, and submit SAS programs
  • view messages about your SAS session and about any SAS programs that you submit
  • browse output from SAS programs

a. Program Editor, Log, Results, SAS Notepad, Output, Explorer
b. Explorer, Results, enhanced Editor or Program Editor, Log, Output
c. File Shortcuts, Explorer, enhanced Editor or Program Editor, Log, Results
d. Results, Output, enhanced Editor or Program Editor, Log, Explorer


You use the Explorer window for managing SAS files; the Results window for navigating and managing output; the enhanced Editor or Program Editor window for working with SAS programs; the Log window for viewing messages; and the Output window for browsing output.  


3.  

 File shortcuts

a. remain in effect until you submit the file.
b. enable you to open, browse, and submit files.
c. are available only for files that were created using SAS.
d. are permanent shortcuts for accessing frequently used files.


A file shortcut is a temporary, optional name that is used to identify an external file (a file that is created and maintained in your host operating environment, but which is not managed by SAS) to SAS. You can use shortcuts to open, browse, and submit files, and you can reference file shortcuts in your SAS programs.  


4.  

 When you define a library,

a. you cannot manage files within it until you delete the library.
b. you must assign a library name only if you use the default engine.
c. you indicate the location of your SAS files to SAS.
d. the path cannot be an existing directory or other storage location.


To indicate the location of your SAS files to SAS, you define libraries. In order to define a library, you must assign a library name, a path, and an engine. The library name, which you can think of as a temporary nickname or shortcut, is a logical name for the physical location that is described by the path. (It's a good idea to create the directory or other storage location before defining the library.) After defining a library, you can manage SAS files within it.  


5.  

 How can you copy a SAS file from one library to another using the Explorer window?

a. Drag and drop the file onto the target library folder.
b. Copy the file shortcut and assign the target library name to it.
c. Select Copy from the pop-up menu for the target library.
d. Click the Copy tool. Then click the target library folder.


In the Explorer window, you can copy a SAS file in three ways. (1) You can drag and drop the file onto the folder for the target library. (2) You can select Copy from the pop-up menu for the file, then select Paste from the pop-up menu for the target library. (3) You can select the file and copy and paste it using either the Edit menu or keyboard shortcuts.  

6.  

 What's the difference between the Results window and the Output window?

a. The Output window displays a tree view of output that is created during a SAS session. The Results window displays the output itself.
b. The Output window displays only output from SAS programs. The Results window displays other types of output.
c. The Results window displays a tree view of output that is created during a SAS session. The Output window displays the output itself.
d. The Results window is available for viewing output only in the Windows environment. The Output window is available in all SAS environments.


The Results window is a tool for managing both basic types of SAS output: listing output (which is displayed in the Output window) and HTML output. From the Results window, you can choose an individual item of listing output to display in the Output window, or you can save and delete individual items of output.  


7.  

 Which of the following is true about SAS output?

a. You can create both listing output and HTML output.
b. You can manage all types of SAS output in the Results window.
c. Listing output and HTML output appear as separate items in the Results window.
d. all of the above


You can create listing output, HTML output, or both types of output. The Results window displays separate icons for the two basic types of output, and it helps you navigate and manage both types of output. 


8.  

 When you delete a library,

a. the contents of the library are deleted.
b. the pointer remains, even if you delete the library's contents.
c. SAS no longer has access to the library's contents.
d. you can manage SAS files within it until the end of the SAS session.


When you delete a SAS library, the pointer is deleted, and SAS no longer has access to the files in the library. However, the contents of the library still exist in your operating environment.  


9.  

 How can you view the properties for a SAS file?

a. In the Results window, double-click the icon for the file.
b. Print the file and view its properties in the Log window.
c. In the Explorer window, select Properties from the pop-up menu for the file.
d. Drag and drop the file onto the enhanced Editor window.


To view the properties for a SAS file, select Properties from the pop-up menu for the file in the Explorer window.  


10.  

 Which of the following tasks can you perform using the Results window?

a. view and print output
b. save individual items of output
c. delete individual items of output
d. all of the above


The Results window helps you navigate and manage output from SAS programs that you submit. You can view, save, and print individual items of output.
