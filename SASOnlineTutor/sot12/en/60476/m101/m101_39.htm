Creating Queries That Accept Parameters

In the Profit query, you displayed all products and their related information. However, you may want to focus on specific categories of products. If you simply add a filter to your query to select a product category, only that category ever appears. Instead, you may want to choose a category when you run the query.

To choose profit information for only one product category at a time, you can create a parameter for your query. Used with a filter in your query, the parameter makes the filter dynamic.

Creating a query that accepts a parameter is a two-step process. You

  1. create a parameter that prompts you to select a value
  2. set a filter based on the parameter.


How Parameters Work

When you create a parameter, you create a special variable called a macro variable. The macro variable stores a user-supplied value (the category that you select, for instance). A filter in your query can reference the value stored in the macro variable. For example, for your profit query you can specify the macro variable &category as a parameter, then set a filter that equals the value of &category.

When you run a query with a parameter, it prompts you to select a value for a column, then displays only rows with matching values for that column. In the case of your profit query, it prompts you to select a product category, then displays only products in the selected category.
