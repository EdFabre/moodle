Using Outer Joins

When you needed to view supplier information for products, you created the Products to Order query to join tables. In that query, you used an inner join to select only the matching rows from the Products and SupplyInfo(Suppliers$) tables. Later, when you manually joined the Orders table to these tables, you again used an inner join.

However, sometimes you may need to use outer joins instead. An outer join combines and displays all rows that match across tables, plus some or all of the rows that do not match. You can think of an outer join as an augmentation of an inner join: an outer join returns all rows generated by an inner join, plus additional (nonmatching) rows.
