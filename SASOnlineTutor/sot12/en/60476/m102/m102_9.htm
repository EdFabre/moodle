Specifying Default Footnote Text

In the Getting Started tutorial, you often deleted the footnote text for the reports that you generated. However, suppose that for your list report, you want the footnote to specify the date that the report was generated. You can specify footnote text in the Options window.

The default footnote for tasks in SAS Enterprise Guide is as follows:

Generated by the SAS System (&_EGSERVERNAME, &SYSSCPL) on %SYSFUNC(DATE(), EURDFDE9.) at %SYSFUNC(TIME(), TIMEAMPM8.)

This footnote text uses special variables (called macro variables) to hold information that is gathered when the task runs. It also uses SAS macro functions and SAS functions to display the date and time when the task runs.


Macro Variable Value
_EGSERVERNAME name of server that SAS Enterprise Guide is using
SYSSCPL name of operating system under which SAS Enterprise Guide is running

Function Action
%SYSFUNC executes a SAS function
DATE returns the current date
TIME returns the current time


In the footnote text, the macro variables _EGSERVERNAME and SYSSCPL hold values for the server and operating system that SAS Enterprise Guide is using. The SAS macro function %SYSFUNC is used to execute another SAS function, DATE(), and then to format the value from the date function with the EURDFDE9. format. Likewise, %SYSFUNC is used to execute the TIME() function and return the value after it is formatted with the TIMEAMPM8. format.

This is an example of a footnote that this footnote text creates in the final report:

Generated by the SAS System (Local, XP_PRO) on 25MAY2004 at 9:56 AM

You can edit the default footnote text, and you can include other macro variables or use other formats to change the appearance of the date and time. For example, suppose that the board members for your specialty foods shop aren't interested in seeing the server name and operating system on your reports. In this case, you can remove the _EGSERVERNAME and SYSSCPL macro variables from the footnote text. On the next page, you'll have a chance to edit the default footnote.

You can learn more about macro variables in the lesson .


Setting Options for Titles

You'll probably also want to specify unique titles for your reports. Default titles for tasks are listed in each individual task window rather than the Options window. For each task, you can either accept the default task-specific title or type in a custom title.

In addition to the title text, some tasks write the SAS procedure name on a separate line in the title that is not editable in the task window. In the report below, notice that the title The FREQ Procedure is included below the custom title.





To prevent SAS Enterprise Guide from displaying the name of the SAS procedure that is used to run the task, you can set an option, as you'll see next.
