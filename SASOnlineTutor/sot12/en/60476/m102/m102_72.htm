Preparing Data for Reports
Before you create reports in SAS Enterprise Guide, you need to add the data for your report to the project. Using SAS Enterprise Guide, you can do any of the following to prepare data for reports:

  • add SAS data sets and other data types (Microsoft Excel, Microsoft Access, dBASE, Lotus 1-2-3, and Paradox files)
  • import data from a text file or Microsoft Excel file
  • use the Query Builder to generate data with only the information that you want to use.

You can add data from your local machine or from any server that you can access.

Setting Options for Results
To produce the list report that you want, you first need to set options to control the appearance of your results. In SAS Enterprise Guide, you can set various options to change the following features of your reports:

  • the format of the report
  • the style of the report
  • the footnote and title text for the report.

You can use the Options window to set a default value for these options and then override the defaults as necessary for individual reports.

You can generate any or all of the following report formats from your tasks: HTML, PDF, RTF, or Text Output. If you specify HTML as a result format, you can also use styles to modify the appearance of your reports. You can select a default style for HTML results by using the Report style drop-down list in the Options window. You can also set the default style by using the Style Manager window. This method enables you to preview the style before you select it.

You can specify footnote text in the Options window. The default footnote text uses special variables (called macro variables) to hold information that is gathered when the task runs. It also uses SAS macro functions and SAS functions to display the date and time when the task runs.

Macro Variable Value
_EGSERVERNAME name of server that SAS Enterprise Guide is using
SYSSCPL name of operating system under which SAS Enterprise Guide is running

Function Action
%SYSFUNC executes a SAS function
DATE returns the current date
TIME returns the current time

You can edit the default footnote text, and you can include other macro variables or use other formats to change the appearance of the date and time.

Default titles for tasks are listed in each individual task window rather than the Options window. For each task, you can either accept the default task-specific title or type in a custom title. You can also suppress the printing of the name of the SAS procedure that is used to run the task.

Creating List Reports
A list report is a detailed report that lists each row or observation in the data set. For this type of report, you can also do the following:

  • group and sort data
  • display column totals
  • format variable values
  • use custom label text for the column headings
  • specify titles and footnotes.

To create a list report, you use the List Data task.

Creating One-Way Frequency Reports
A one-way frequency report describes your data by showing the distribution of a variable's values. You use the One-Way Frequencies task to generate frequency tables from your data. You can also perform binomial and chi-square tests by using this task.

Creating Summary Reports
A summary report displays descriptive statistics in tabular format for some or all of the columns in your data. To create a summary report, you use the Summary Tables task. For this type of report, you can also customize the following:

  • table layout
  • statistics that are used
  • format of the variable values
  • headings for columns and rows
  • report titles and text in the box area.

Creating Graphs
Using SAS Enterprise Guide, you can easily create graphs including the following:

  • bar charts
  • pie charts
  • area charts
  • line charts
  • scatter charts
  • surface charts
  • map charts
  • bubble plots
  • finance charts
  • donut charts
  • contour plots
  • radar (or star) charts.

Setting Options for Graphs
For the graphs you create, you can set the image format and other options. The format that you select depends on how you plan to use the graph. ActiveX is the default image format for SAS Enterprise Guide.

There are additional options that you can set for various image formats. For example, if you are generating GIF or JPG format graphs, you can set options to include titles and footnotes as part of the image.

If your graph will be viewed in the Microsoft Windows environment, ActiveX is a very versatile format to use. The ActiveX image format has the following features:

  • The graph is interactive—you can change many of the features of the graph without having to rerun the task.
  • The graph, including any changes that you make by using the interactive features, can be saved as a JPG or BMP file.
  • You can copy the graph from SAS Enterprise Guide and paste it directly into Microsoft PowerPoint. During a presentation you can use the interactive features of the graph.
  • You can copy the graph from SAS Enterprise Guide and paste it directly into Microsoft Word as an image.

Combining Reports
To combine results in a document, you generate RTF output for the individual reports that you want to combine and then open them in any word processing program (such as Microsoft Word) that will open a document that is saved as a Rich Text Format (RTF) file. In addition, you can copy and paste ActiveX object graphs directly into Microsoft Word.

By default, the result format is set to HTML. The Properties window for each task enables you to override the default settings and select result formats, an image format, or a new style to apply to the results for the task individual task.

You can also use the Document Builder to create one HTML file from several HTML results that are produced in SAS Enterprise Guide. To customize the appearance of your HTML file, you can select a style to apply to the document. After you have the results that you want in one HTML file, you can edit the HTML file to add titles, text, and so on.

Using Styles to Customize HTML Reports
To further customize the look of your HTML reports, you can also do the following:

  • modify the predefined styles that SAS Enterprise Guide provides for you
  • create your own new styles from existing styles
  • add an external style that resides on a Web server or elsewhere on your network.

You use the Edit Style window to modify and create styles. In the Edit Style window, you can select style elements graphically or from a drop-down list. To customize styles, you can define text and border attributes, specify background and border images, and create your own custom properties. As you edit styles, the Edit Style window displays your changes graphically.

Creating and Applying Custom Formats
To write variable values in predefined ways, you can create and apply formats. You can use predefined SAS formats, or you can create your own formats, especially if you have data that includes codes for variable values. When you create a custom format, you can apply the format to any data sets that contain the appropriate values to be formatted.

Using Traffic Lighting in a Summary Table
To apply a style to a table cell based on the value of the data in that cell, you can use traffic lighting. Traffic lighting is a visual cue that often uses the colors of a traffic light: red, yellow, and green. When you use traffic lighting, you can see at a glance which values are within specifically defined ranges.

To add traffic lighting to table cells in a summary table, you do the following:

  • create a custom format
  • create the summary table that you want
  • modify the summary table code to add the format.

To modify the summary table code, you add the task as a code template, then add a STYLE= option to a variable or statistic in the table. The code for adding a background color that is based on a format is as follows:

*{style={background=format.}}

Additional Features
You can create and edit your own custom style attributes using the Custom tab in the Edit Styles window.

Points to Remember

  • If the data that you are using is not stored on the server where the task runs, SAS Enterprise Guide copies the data to the server before it runs the task. Keep in mind that this can be time-consuming for large data sets.

  • Although you can choose to create more than one result format for each task that you run, it takes time to create each report. The more formats that you create, the longer your task will take to process.

  • To generate crosstabulation (contingency) tables, you can use the Table Analysis task. A crosstabulation table summarizes for two or more classification variables by showing the number of observations for each combination of variable values.

  • An alternative to creating a format is to replace values in a column by using a query. The advantage of creating a custom format is that you can apply the format to other data sets.

  • User-defined formats always associate variable values with character values. If you attempt to perform arithmetic operations on user-formatted numeric values, the unformatted values are used.
