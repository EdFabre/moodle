Preparing Data for Reports

Your first report will be a list report of products that are sold in the specialty foods shop. However, before you create reports in SAS Enterprise Guide, you need to add the data for your report to the project. Using SAS Enterprise Guide, you can do any of the following to prepare data for reports:

  • add SAS data sets and other data types
  • import data from a text file or Microsoft Excel file
  • use the Query Builder to generate data with only the information that you want to use.


You have learned to perform these tasks in the Getting Started tutorial. This topic simply reviews the main ways of getting data into SAS Enterprise Guide.


Review of Adding and Importing Data

One of the benefits of SAS Enterprise Guide is that it makes it easy for you to access data. You can select InsertData and browse to add any of the following data types that you want to use for your reports:

  • SAS data sets
  • Microsoft Excel and Access files
  • dBASE files
  • Lotus 1-2-3 files
  • Paradox files.

You can add data from your local machine or from any server that you can access. If the data that you are using is not stored on the server where the task runs, SAS Enterprise Guide copies the data to the server before it runs the task. Keep in mind that this can be time-consuming for large data sets.

You can use the Import Data task to create a SAS data set from a text file or a Microsoft Excel file. To import data, you select ToolsImport Data and then go through the steps in the task using the Next button. You can create a report directly from a Microsoft Excel file, but to use a text file for a report you must first create a SAS data set from it.
