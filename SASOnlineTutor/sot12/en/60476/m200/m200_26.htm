Detecting and Resolving Common Problems (continued)

Unbalanced Quotation Marks

Some syntax errors, such as the missing quotation mark after HIGH in the program below, cause SAS to misinterpret the statements in your program. 

     data sasuser.admitfee;
        set sasuser.admit;
        where actlevel='HIGH;
     run;
     proc print data=sasuser.admitfee;
        var id name actlevel fee;
     run;

The first clue that something is wrong is in the Code Editor window. Normally text inside quotation marks is magenta in color. Because the ending quotation mark is missing, all the text after the first quotation mark is magenta.






When you submit a program with unbalanced quotation marks, SAS is unable to resolve the DATA step, and "Running DATA STEP" appears in the Task Status window.

Canceling the task, editing, and resubmitting the program does not solve the problem. SAS still considers the quotation marks to be unbalanced. This is one syntax error that you should learn to detect before you submit your program to SAS. If you submit the program with unbalanced quotation marks, you will have to exit SAS Enterprise Guide and then start a new session.

Resolving the Problem

  1. In the Task Status window, right-click the code and select Delete. This cancels the program.
  2. Edit the code to add the missing quotation mark.
  3. Submit the code and overwrite the previous results.
