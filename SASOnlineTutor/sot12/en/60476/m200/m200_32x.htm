1.  

 As you write and edit SAS programs it's a good idea to

a. begin DATA and PROC steps in column 1.
b. indent statements within a step.
c. begin RUN statements in column 1.
d. do all of the above


Although you can write SAS statements in almost any format, a consistent layout enhances readability and enables you to understand the program's purpose. It's a good idea to begin DATA and PROC steps in column one, to indent statements within a step, to begin RUN statements in column one, and to include a RUN statement after every DATA step or PROC step. 

2.  

 After you have submitted a program, what indication do you have that there were errors in the program?

a. A red X appears on the code item in the Project window.
b. The log displays the word ERROR with an explanation of the error.
c. The log displays the word ERROR and identifies a possible location of the error.
d. all of the above


Information is written to the log for every program that SAS executes. When there is an error, the log displays the word ERROR, identifies the possible position of the error, and gives an explanation of the error. In addition, a red X appears on the code item in the Project window. 


3.  

 What is the first problem you should notice and resolve in the following program? 
     proc sort data=clinic.stress out=maxrates;
     by maxhr;
     run;
     proc print data=maxrates label double noobs;
        label rechr='Recovery Heart Rate;
        var resthr maxhr rechr date;
        where toler='I' and resthr>90;
        sum fee;

a. There should be a RUN statement at the end of the program.
b. The color-coding indicates that there are unbalanced quotation marks.
c. Although there is no RUN statement, this program is fine to submit in SAS Enterprise Guide.
d. The BY statement is not indented properly.


The missing quotation mark in the LABEL statement causes the Code Editor window to color the code magenta until the next quotation mark after toler=. There is another unbalanced quotation mark starting after the I. It is important to fix this error before submitting the program to SAS. 


4.  

 What generally happens when a syntax error is detected?

a. SAS continues processing the step.
b. SAS continues processing the step, and the Log window displays messages about the error.
c. SAS stops processing the step in which the error occurred, and the Log window displays messages about the error.
d. SAS stops processing the step in which the error occurred, and messages about the error appear in the HTML results.


Syntax errors generally cause SAS to stop processing the step in which the error occurred. When a program that contains an error is submitted, messages regarding the problem also appear in the Log window. When a syntax error is detected, the Log window displays the word ERROR, identifies the possible location of the error, and gives an explanation of the error.  


5.  

 A syntax error occurs when

a. some data values are not appropriate for the SAS statements that are specified in a program.
b. the form of the elements in a SAS statement is correct, but the elements are not valid for that usage.
c. program statements do not conform to the rules of the SAS language.
d. none of the above


Syntax errors are common types of errors. Some SAS system options and features of the code editing window can help you identify syntax errors. Other types of errors include data errors, semantic errors, and execution-time errors.  

6.  

 How can you tell whether you have specified an invalid option in a SAS program?

a. A log message indicates that the option was ignored and SAS continued to process the program without it.
b. A log message indicates that another option was expected for the statement.
c. The message "PROC running" or "DATA step running" appears at the top of the active window.
d. You can't tell until you view the output from the program.


When you submit a SAS statement that contains an invalid option, a log message notifies you that the option is not valid or not recognized. You should recall the program, remove or replace the invalid option, check your statement syntax as needed, and resubmit the corrected program. 


7.  

 What methods can you use to open a SAS program that you have saved in your operating environment?

a. Drag and drop the file from Windows Explorer to the Project window.
b. Select Insert Code and locate the file on the Existing tab.
c. In Windows Explorer, right-click the SAS program and select Open with Enterprise Guide.
d. all of the above


To open a stored program, you can use the Insert Code window, the Files folder in the Server List, and Windows Explorer. 


8.  

 Which statement about the RUN statement is false?

a. When SAS processes a program with a missing RUN statement at the end of a program, that last step in the program does not execute until a RUN statement is added.
b. SAS Enterprise Guide automatically adds a RUN statement to code you submit. If you omit the RUN statement in your code, SAS uses the RUN statement added by SAS Enterprise Guide.
c. Because SAS Enterprise Guide adds a RUN statement to the code that you submit, it is better not to add an extra RUN statement in your program.
d. SAS recognizes the end of the current step when it encounters a DATA or PROC step, or a RUN or QUIT statement.


Although SAS Enterprise Guide appends a RUN statement as a precaution so that your program will execute if you omit it, you should always add a RUN statement to end of each step in your program. 


9.  

 Which of the following programs contains a syntax error?

a. proc sort data=sasuser.mysales;
   by region;
run;
b. dat sasuser.mysales;
   set mydata.sales99;
run;
c. proc print data=sasuser.mysales label;
   label region='Sales Region';
run;
d. none of the above


The DATA step contains a misspelled keyword (dat instead of data). However, this is such a common (and easily interpretable) error that SAS produces only a warning message, not an error.  


10.  

 What does the following log indicate about your program?

proc print data=sasuser.cargo99
   var origin dest cargorev;
   22
   76
ERROR 22-322: Syntax error, expecting one of the following: 
              ;, (, DATA, DOUBLE, HEADING, LABEL, 
              N, NOOBS, OBS, ROUND, ROWS, SPLIT, STYLE, 
              UNIFORM, WIDTH.
ERROR 76-322: Syntax error, statement will be ignored.
11         run;

a. SAS identifies a syntax error at the position of the VAR statement.
b. SAS is reading VAR as an option on the PROC PRINT statement.
c. SAS stopped processing the program because of errors.
d. all of the above


Because there is a missing semicolon at the end of the PROC PRINT statement, SAS interprets VAR as an option on PROC PRINT and finds a syntax error at that location. SAS stops processing programs when it encounters a syntax error.
