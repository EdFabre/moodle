Detecting and Resolving Common Problems

In addition to correcting spelling mistakes, you might need to resolve several other types of common syntax errors. These errors include

  • omitting semicolons
  • specifying invalid options
  • leaving quotation marks unbalanced.

Another common problem is omitting a RUN statement at the end of a program. Although this is not technically an error, it can produce unexpected results if you submit code from within SAS. For the sake of convenience, we'll consider this error together with syntax errors.

The table below lists these problems and their symptoms.


Problem Symptom
missing semicolon log message indicates an error in a statement that seems to be valid
invalid option

log message indicates that some other option was expected on the statement

unbalanced quotation marks the code does not execute and a message in the Task Status window indicates that the DATA step is running
