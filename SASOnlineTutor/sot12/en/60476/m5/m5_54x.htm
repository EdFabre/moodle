1.  

 Which program creates the following table?

  ActLevel All Levels
HIGH LOW MOD
Fee Sum 844.30 844.30 998.35 2686.95

a. 
proc tabulate data=clinic.admit;
   class actlevel;
   var fee;
   table fee sum,actlevel all;
   label all='All Levels';
run;
b. 
proc tabulate data=clinic.admit;
   var actlevel;
   class fee;
   table actlevel all fee;
   keylabel all='All Levels';           
run;
c. 
proc tabulate data=clinic.admit;
   class actlevel;
   var fee;
   table fee,actlevel all;
   keylabel all='All Levels';           
run;
d. 
proc tabulate data=clinic.admit;
   class actlevel;
   var fee;
   table actlevel all,fee;
   keylabel all='All Levels';           
run;


The output is a two-dimensional table in which the KEYLABEL statement is used to label the special variable ALL. The CLASS statement specifies that values of ActLevel be used as categories, and the VAR statement specifies that Fee be used for analysis. The TABLE statement specifies ActLevel and ALL in the column dimension, and it specifies Fee in the row dimension. 

2.  

 Which TABLE statement correctly requests statistics to show average sales?

a. 
table region,sales(mean),year;
b. 
table year salesrep*sales*average;
c. 
table salesrep*year,sales mean;
d. 
table region,year,sales*mean;


To request a statistic, you use the asterisk (*) operator to attach the statistic to the variable. The statistic MEAN requests the average for the variable Sales.

4.  

 If the error message Variable VARIABLE in list does not match type prescribed for this list is generated, what has happened?

a. You specified a numeric variable as a class variable.
b. You specified a character variable as an analysis variable.
c. You forgot to specify a variable as either class or analysis.
d. You specified a variable as both class and analysis.


Class variables can be character or numeric, but analysis variables must be numeric. If you specify a character variable as an analysis variable, the "does not match type" message is generated. 


5.  

 Which type of table does this TABLE statement create? 
     table dest*n orig*n boarded*pctsum 
           transferred*pctsum deplaned revenue;

a. a one-dimensional table
b. a two-dimensional table
c. a three-dimensional table
d. The TABLE statement is incorrectly specified.


If a TABLE statement doesn't contain a comma, it requests a one-dimensional table, no matter how many variables or statistics it specifies. A TABLE statement that contains one comma specifies a two-dimensional table. Two commas indicate a three-dimensional table.

7.  

 Which PROC TABULATE step below is correct?

a. 
proc tabulate data=finance.loans;
   class customertype;
   table customertype n,pctn;          
run;
b. 
proc tabulate data=finance.loans;
   class customertype;
   var customertype;
   table customertype,n;          
run;
c. 
proc tabulate data=finance.loans;
   var customertype;
   table customertype amount,n;          
run;
d. 
proc tabulate data=finance.loans;
   class customertype;
   table customertype,n pctn;          
run;


Statistics can be specified in any dimension, but they must all be in the same dimension, so a is incorrect. Each variable in the TABLE statement must be specified in either a CLASS statement or a VAR statement (but not in both), so b and c are incorrect. 


8.  

 Why is the following PROC TABULATE step incorrect? (Survive and Shock are character variables.) 
proc tabulate data=clinic.heart;
   var survive shock;
   table survive shock,sum pctsum;
run;

a. The wrong statistics are requested for the variables.
b. The variables are specified as analysis variables instead of as class variables.
c. The variables and statistics are specified in different dimensions.
d. only a and b above


Class variables can be character or numeric, but character variables must always be class variables. If you specify only class variables in your TABLE statement, the only statistics you can request are N and PCTN (percent of total frequency). Variables and statistics can be specified in any dimension, but all statistics must be in the same dimension. 


9.  

 What happens when you submit the program below? 
proc tabulate data=clinic.admit;
   class actlevel sex;
   table actlevel,sex;
   where actlevel='HIGH';
run;

a. The table has only one column.
b. The table has only one row.
c. The table has two columns and two rows.
d. The program causes error messages and stops processing.


Because the WHERE statement selects only one value of ActLevel (which is in the row dimension), the table contains only one row.
