Setting Up a Table

The key to using PROC TABULATE is planning.

Unlike PROC PRINT, PROC TABULATE doesn't create default reports. So, before you begin writing a PROC TABULATE step, it's a good idea to sketch the table you want.



After sketching the table, you can write the basic code to compute the statistics that you want to display.

When you are satisfied with the basic report, you can modify its appearance by adding options and statements. In fact, once you define the basic structure of your table, enhancing the table is easy. 



  

    
 
      So, how do you begin to sketch your table? 
        
 In order to set up a table with PROC TABULATE, you need to identify 
          the data you are analyzing, and then determine 
        
    
          
  •  which variables, if any, you need for classifying your data 
          
  •  which variables, if any, you need for analyzing your data 
          
  •  the type of table you need for representing your data. 
        

        
 Try to visualize a table that shows all the patients that have been 
          admitted to a wellness clinic. The table should include
        
    
          
  •  columns that display the average age for patients 
          
  •  rows that display categories for different levels of patient activity.
