Understanding SAS Enterprise Guide Software and SAS Software
SAS Enterprise Guide is a Windows client application that serves as an interface to SAS. In this lesson and other SAS programming lessons, you use SAS Enterprise Guide to write SAS programs and submit them to SAS for processing. When SAS finishes processsing the code, it sends the results and log messages back to SAS Enterprise Guide to display.

Using the Main SAS Enterprise Guide Windows
You use the main SAS Enterprise Guide windows to explore and manage your files, and to start and monitor tasks.

Use this window ... To ...
Project
  • manage your work
  • delete, rename, rearrange items
Task List
  • add items to a project (such as code and data)
  • begin tasks
Task Status
  • monitor the status of code as it is processed by SAS

SAS windows have many features that help you get your work done. For example, you can

  • maximize, minimize, and restore windows
  • use pull-down menus, pop-up menus, and toolbars
  • dock and undock windows
  • get more help
  • customize toolbars, pop-up menus, and icons.

Viewing SAS Libraries and Files
You use the Server List to view SAS libraries and the SAS data sets in libraries. To view the contents of a SAS data set, you add it to the project and the data opens in the data grid. You can view properties for the SAS data set by right-clicking the data set in the Project window and selecting Properties from the shortcut menu.

Writing and Submitting Code
You write, edit, and submit programs in the Code Editor window. After SAS processes the program, the Log contains messages about processing. Also, any results are generated and they open automatically in SAS Enterprise Guide.

Managing Your Workspace
There are options you can set in SAS Enterprise Guide to customize your workspace. In addition, there are various things you can do to make your projects and code more manageable.

Getting Help
Be sure to take advantage of the different types of online help that are available from the Help menu and the SAS Web.

Points to Remember
  • When you run code, SAS Enterprise Guide sends the code to SAS for processing and SAS sends back the log and any results that are generated by procedure code.

  • When you save your project, the code, results, and pointers to data are saved in a single file.

  • When you submit code to SAS, SAS Enterprise Guide adds code to create HTML output and set options. Some of the messages in the log refer to the code that SAS Enterprise Guide has added to your code.
