Understanding SAS Enterprise Guide and SAS Software

SAS Enterprise Guide is a Windows client application that serves as an interface to SAS software. In this lesson and other SAS programming lessons, you use SAS Enterprise Guide to write SAS programs and submit them to SAS for processing. When SAS finishes processsing the code, it sends the results and log messages back to SAS Enterprise Guide to display.

SAS can either be on the same computer with SAS Enterprise Guide or it can be on a remote server. The connection between SAS Enterprise Guide and SAS is set up by your administrator. In the SAS Learning Edition, SAS and SAS Enterprise Guide are both installed on your computer and many of the settings are preconfigured.

Starting a SAS Session

Although it is not technically accurate, you can think of a SAS session starting and ending when you start and end SAS Enterprise Guide. In reality, SAS Enterprise Guide does not establish a connection with SAS until there is a need to do so. In this training, a SAS session starts when you submit code to SAS, when you add data to a project, or when you open the server in the server list. You will learn about these tasks later in this lesson.

The first time you submit a SAS program, it will take slightly longer to process because the connection with SAS is being established. Programs that you submit subsequently will process faster.
