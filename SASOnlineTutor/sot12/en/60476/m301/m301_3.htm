Touring the Programming Windows

When you first start SAS Enterprise Guide, the Welcome window opens. In this window you either open an existing project file or create a new project. You'll learn more about projects in the next topic. For now, let's assume that you are starting a new project.

When you start a new project, you get the default SAS Enterprise Guide workspace layout with the following windows open and docked: Project, Task List, and Task Status.



If you are familiar with using the SAS windowing environment (SAS Explorer, Enhanced Editor, Output and Log windows) for programming, you might be wondering how you will write and submit programs in this environment. In addition to the windows that are already open, you will open the Code Editor and Server List. You can view the Log and Results windows after you submit a program.

The table below lists the windows that you use to accomplish your programming tasks in SAS Enterprise Guide.


Task SAS Windowing Environment (DMS) SAS Enterprise Guide
Write and submit programs Enhanced Editor or Program Editor Code Editor
View Libraries SAS Explorer Server List
View SAS Listing output Output window Output window. Text output is not created by default. It is available by selecting an option in the Options window. You can open the Output window after a program runs.
View system messages Log window Log window. The Log is available after the program runs.
Navigate results Results window Project window. Results from each program are organized in the project under the Code item that created the results.
View data VIEWTABLE Data grid


All windows in SAS Enterprise Guide have features that you are probably familiar with. You can do the following:

  • Maximize, minimize, close, restore, and reposition windows.
  • Use main menus, pop-up menus, and toolbars.
  • Get more help.
For more information about using windows, select HelpEnterprise Guide Help and read the topics in Using the Workspace.
