Touring the Programming Windows (continued)

The Server List

You can use the Server List to do the following:

  • view a list of all the available SAS servers
  • view SAS libraries on particular servers
  • view and access SAS data sets
  • view and access local and remote files.

The Server List does not open automatically the first time you start SAS Enterprise Guide. You can open it by clicking the Server List tool () on the toolbar or by selecting ViewServer List. By default, the Server List is docked on the right side of the workspace.





The servers that are listed in the Server List are the servers that have been registered in Enterprise Guide Administrator. If you have SAS on your local computer, the server name is Local in the Server List. In some installations there could be many servers listed in this window.

If you have access to more than one server, you can see and change the default server by selecting ToolsAdministrator Options.


Viewing SAS Libraries

The libraries that are currently defined for your SAS session are listed under Libraries. As usual, SAS defines several libraries when the SAS session begins: Sashelp, Sasuser, and Work. In addition, SAS Enterprise Guide can define libraries when it starts.

To view the SAS data sets in a library, do the following:

  1. In the Server List, double-click the server name to open the server.

    The first time you double-click a server to open it, it might take some time if SAS Enterprise Guide has to establish a connection with SAS.



  2. Double-click the Libraries folder.

  3. Double-click the library name (or right-click the library name and select Open from the pop-up menu). A list of the SAS data sets that are contained in the library is displayed.


You cannot view the properties of a SAS data set by using the Server List. You must double-click the file and add it to the project before you can view properties. You learn more about adding data to a project later in this lesson.
