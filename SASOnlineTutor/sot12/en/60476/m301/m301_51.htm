Understanding SAS Enterprise Guide Software and SAS Software
SAS Enterprise Guide is a client application that uses the processing power of SAS software. You can use SAS Enterprise Guide to write SAS programs and submit them to SAS for processing. When SAS finishes processing the code, it sends the results and log messages back to SAS Enterprise Guide to be displayed.

Touring the Programming Windows
When you first open SAS Enterprise Guide, you see the Project window, Task List, and Task Status window open in the workspace. For SAS programming, you will also use the Server List and the Code Editor window. Although the environment differs from the SAS windowing environment, you will find windows in SAS Enterprise Guide to accomplish your programming tasks.

Writing and Submitting Code
In SAS Enterprise Guide you use the Code Editor to write and submit SAS programs. After SAS finishes processing your program, you can view the log by double-clicking Log in the Project window. If you need to edit and resubmit your program, you will be prompted whether or not to overwrite the previous results. If you choose not to overwrite the results, a new item for the edited code is added to the project.

Storing and Accessing Code
When you write a SAS program by using SAS Enterprise Guide, the program is embedded in the project file. You can export a copy of the program to a file to save it. You can also create a pointer to the saved code so that the project file contains a shortcut to the saved file rather than the file itself. You can open a stored SAS program by selecting Insert Code. This creates a shortcut to the file. You can change the properties of the file to embed it in the project if you don't want your changes to affect the file.

Working with Results
SAS Enterprise Guide enables you to select result formats (HTML, PDF, RTF, text) to produce from your programs. You can specify result formats for one program by using the Preferences window for that program. You can specify default result formats for all the programs that you run by using the Options window. You can also specify the style of your HTML results.

Working with Data
SAS Enterprise Guide makes it easy to access data. After you add data to your project, you can view the properties by right-clicking the data in the Project window and selecting Properties from the pop-up menu. You can add and open SAS data sets and edit them in the data grid. If the data that you add is not located in an active library, SAS Enterprise Guide submits a LIBNAME statement for you.

In addition to SAS data sets, you can also add other types of data. You can use this data as input to tasks and queries, but you cannot modify the data. You can use the Import Data task to create a SAS data set from a Microsoft Excel file or a text file.

Generating and Editing Task Code
SAS Enterprise Guide has point-and-click tasks for many of the SAS procedures that you use in your programs. You can use these tasks to quickly and easily generate SAS programs. If the task window does not have all the options that you want to use, you can edit the generated code and submit it. There are three ways that you can edit task code:

  • Edit a copy of the generated task code.
  • Insert lines of code into the task code.
  • Add code before or after all program or task code that runs in SAS Enterprise Guide.

Generating and Editing Query Code
In SAS Enterprise Guide, you can use the point-and-click Query Builder to generate SQL code that you can edit. When you edit code generated from the Query Builder, the code is separated from the task.

Points to Remember
  • When you run code, SAS Enterprise Guide sends the code to SAS for processing and SAS sends back the log and any results that are generated by procedure code.

  • When you save your project, the code, results, and pointers to data are saved in a single file.

  • When you submit code to SAS, SAS Enterprise Guide adds code to create HTML output and to set options. Some of the messages in the log refer to the code that SAS Enterprise Guide has added to your code.

  • When you run a task on data, it is most efficient if the data is located on the same server where the program is running. If the data is remote, SAS copies the data to the server where the program will run.
