Writing and Submitting SAS Code (continued)

Editing and Debugging SAS Code

After you submit a program, you can reopen the code that you have submitted by double-clicking the item for the code in the Project window. Then you can edit the code in the Code Editor. If the code that you submitted had errors, two things happen:

  • A red X appears on the Code item in the Project window.
  • The Log window automatically opens so that you can read the error messages.

For most errors you can open the code, edit it, and resubmit it. See the following table for suggestions about using the Code Editor to find syntax errors.


To find... Do this...
Undefined or misspelled keywords On the Editor tab of the Options window, click the Appearance button. Set the file elements Defined keyword, User defined keyword, and the Undefined keyword to unique color combinations.

When SAS recognizes a keyword, the keyword changes to the defined colors. You'll be able to easily spot undefined keywords by looking for the colors that you selected for undefined keywords.

Unmatched quotation marks Look for one or more lines of the program that are the same color.

Text that follows a quotation mark remains the same color until the string is closed with a matching quotation mark.
If you submit a program that contains unbalanced quotation marks, you must right-click the code in the Task Status window and select Delete to cancel the task. Then you can edit and resubmit the code.

Unmatched comments Look for one or more lines of the program that are the same color.

Text that follows an open comment symbol ( /* ) remains the same color until the comment is closed with a closing comment symbol ( */).

Matching DO-END pairs Place the cursor within a DO-END block and press Alt+[.

The cursor moves first to the DO keyword. If one of the keywords is not found, the cursor remains as positioned.

When both of the keywords exist, pressing Alt+[ moves the cursor between the DO-END keywords.

Matching parentheses or brackets Place the cursor on either side of the parenthesis or bracket. Press Ctrl+].

The cursor moves to the matching parenthesis or bracket. If one is not found, the cursor remains as positioned.

Missing semicolons ( ; ) Look for keywords that appear in normal text.
